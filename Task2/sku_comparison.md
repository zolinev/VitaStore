#Сопоставление товаров

###Автоматическое сопоставление товаров

Для увеличения процента автоматического сопоставления товаров реализуем такой подход:

1. Создаем в Облаке сервис сопоставления товаров с собственной БД. Сервис включает в себя web-интерфейс для технолога, бизнес-логику по сопоставлению товаров и свою собственную БД. В состав БД входит:
	- локальная копия справочника товаров из ERP
	- справочник частых сокращений: (д/рук, д.рук -> для рук; шмп, шамп. -> шампунь)
	- Таблица заявок на ручной разбор
2. При обработке каждого товара из нового магазина сервис нормализует строку с учетом справочника сокращеий + убирает все спец. символы
3. Находит товары, наиболее близки по векторному сопоставлению
4. Если нашелся один товар с совпадением >=95%, то сервис считает автоматическую обраотку успешной и сохраняет ее в ERP
5. Если товар не нашелся или таких товаров больше одного, то сервис создает заявку на ручной разбор

###Ручной разбор
1. Во время загрузки товаров из нового магазина товаровед может следить за прогрессом в Веб-интерфейсе
2. Если сервис сопоставления товаров не смог автоматически сопоставить товар, товаровед видит задачу на ручной разбор в том же веб-интерфейсе
3. Для ускорения сервис сразу предлагает товароведу ТОП-3 варианта по совпадению
4. Товаровед выбирает подходящий вариант (если из ТОП-3 не подходит, то выбирает из всей БД)
5. После того, как товаровед указал соответствие, сервис считает обработку успешной и сохраняет этот факт в ERP
